# Product

## Register

product

## Users

Developers reviewing Git diffs from a terminal workflow. They use the browser UI when terminal output is too cramped, but they still expect speed, density, keyboard-friendly controls, and trustworthy file navigation.

## Product Purpose

cli-diff opens a local browser interface for Git diff output. It exists to make file-by-file review easier without leaving command-line context. Success means changes are scannable, states are obvious, and controls stay out of the way until needed.

## Brand Personality

Precise, quiet, technical. The interface should feel like a dependable developer tool: compact, legible, and familiar, with polish coming from alignment and clear state rather than decoration.

## Anti-references

Avoid marketing-page styling, oversized empty surfaces, novelty navigation, heavy gradients, decorative motion, and low-density SaaS dashboard patterns. Do not make the app feel detached from Git, code review, or terminal-adjacent workflows.

## Design Principles

- Preserve review flow: navigation, viewing, and settings should never interrupt reading the diff.
- Make state explicit: selected, viewed, collapsed, loading, and empty states need clear affordances.
- Favor earned density: keep information compact while maintaining target size, contrast, and keyboard access.
- Align with the existing system: use shared tokens and primitives before adding one-off styling.

## Accessibility & Inclusion

Target WCAG AA contrast for text and controls. Preserve visible focus indicators, logical keyboard navigation, reduced-motion support, semantic labels, and readable compact text across light and dark themes.
